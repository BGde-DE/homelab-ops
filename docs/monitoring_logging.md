# Monitoring & Logging (Ist / Zielbild)

## Ist (Minimal-Setup)
- Zentrale Logs/Status werden zur Fehlersuche genutzt (Firewall/Proxy/Host Logs).
- Alerts erfolgen pragmatisch (je nach Service).

## Zielbild (Ausbauoptionen)
- Metriken + Dashboards (z. B. CPU/RAM/Disk/Net)
- Backup-Job-Status als Alert (“letzter Erfolg < 24h”)
- Proxy/IDS Events im Blick (Trend statt nur Einzelfall)
